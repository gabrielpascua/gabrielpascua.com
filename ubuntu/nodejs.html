Installing Node 4.0.0

A stable Node 4.0.0 was just released 3 days ago. This made me happy. If you know the history between nodejs and iojs then you will be excited about the future of node too. I'm too lazy to go over node's installation notes and I'm already familiar with nvm so I'll just use that. It might come handy too since node 4 is still new. Let the chips fall where they may if something goes wrong. After everything I've been through, I think I can google my way out of a node prolem.

  1. $ sudo -i
  2. Remove any node installation in your system:
    $ dpkg --get-selections | grep node
    $ apt-get remove --purge node
  3. $ apt-get install nodejs
  4. $ apt get install npm
  5. Install nvm globally to manage node versions:
    $ curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.26.1/install.sh | NVM_DIR=/usr/local/nvm bash
  6. Locate the node version you want to install:
    $ nvm ls-remote
  7. Install it:
    $ nvm install [NODE VERSION]
  8. Set a default node version:
    $ nvm alias default [NODE VERSION]
  9. Add this to your .bashrc file to pick up the system node then restart bash:
    > export NVM_DIR="/usr/local/nvm" > [ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh" # This loads nvm
  10. Verify your node version:
    $ node --version
  11. Troubleshooting:
    • Some depencies refer to your /usr/bin/node version
      $ rm -r /usr/bin/node
      $ ln -s /usr/local/nvm/versions/node/[NODE VERSION]/bin/node /usr/bin/node
      $ rm -r /usr/bin/npm
      $ ln -s /usr/local/nvm/versions/node/[NODE VERSION]/bin/npm npm

      Original node: /usr/bin/nodejs
      Original npm: /usr/lib/node_modules/npm/bin/npm-cli.js
